New Matchup Transition — 3 Approaches

Covers both triggers: completing all 3 votes AND flagging a bad voice

Two Triggers, Two Messages

The transition animation is the same for both cases, but the confirmation text differs:

  • All 3 votes cast: "Vote recorded — New matchup"
  • Bad voice flagged: "Voice flagged — New matchup"
A

Card Sweep Animation

On 3rd vote: Voice cards + vote rows slide out left/right → "Vote recorded" flashes center → new cards slide in. ~800ms total.

On flag: Same sweep animation, but the flagged card gets a brief red flash/shake before sliding out → "Voice flagged" flashes center → new cards slide in. The red flash gives immediate feedback that the flag action was received before the transition begins.

Pros

  • Clear visual break — old content physically leaves
  • Red shake on flag feels responsive and distinct from voting
  • The two triggers feel intentionally different

Cons

  • Most complex to implement (exit/enter choreography + flag variant)
  • ~800ms+ delay, longer with the flag shake
B

Fade-Through with Confirmation Toast

On 3rd vote: Content fades out (~200ms) → toast pill appears: "Vote recorded ✓ — New matchup" → new content fades in (~300ms). ~500ms total.

On flag: Same fade-through, but toast changes to "Voice flagged ⚑ — New matchup" with a red-tinted accent instead of the default. Optionally, the flagged card border briefly flashes red before the fade begins to give instant feedback on which card was flagged.

Pros

  • Fast and clean (~500ms)
  • Toast text handles both messages naturally
  • Red-tinted toast for flags is a subtle but clear distinction
  • Simplest to implement — same animation, different toast content/color

Cons

  • Less dramatic distinction between vote vs flag
  • Subtle — some users might still miss it
C

Full-Screen Flash + Scale Reset

On 3rd vote: Purple/cyan glow pulse washes over content → cards scale down (0.95) then back up (1.0) with new data → matchup counter increments.

On flag: Instead of the purple/cyan pulse, a red glow pulse washes over → same scale bounce → counter increments. The color shift makes it immediately obvious this was a flag, not a vote.

Pros

  • Unmissable — full-screen color tells you exactly what happened
  • Red vs purple/cyan is an instantly readable distinction
  • Gamified feel with the matchup counter

Cons

  • Could feel flashy/fatiguing after many rounds
  • Full-screen color pulses may feel heavy on mobile
  • Red flash could feel punitive (flagging should feel neutral)

My Recommendation

Option B (Fade-Through + Toast) remains the best balance. The same fade animation works for both triggers — only the toast text and accent color change. This keeps the implementation simple (one animation path, parameterized message) while clearly communicating what happened in both cases. The optional red border flash on the flagged card adds just enough distinction without a separate animation system.
